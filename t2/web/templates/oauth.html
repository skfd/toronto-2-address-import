Setup

One-time per OSM server (the dev sandbox and real OSM each have their own application registry, so a prod run needs a separate registration).

1. Register an OAuth2 application

Log into the target server and register a new application:

Open {{ osm_target.label }} OAuth2 applications ↗

  • Name: anything (e.g. t2-address-import-dev).
  • Redirect URI: {{ status.redirect_uri }} — must match exactly.
  • Confidential application: yes (we use a client secret).
  • Permissions: tick read_prefs (read user preferences), write_api (modify the map), write_changesets (comment on changesets).

Save, then copy the resulting Client ID and Client Secret.

2. Fill .env at the repo root

Copy .env.example to .env and fill in:

OSM_API_BASE={{ status.api_base }}
OSM_CLIENT_ID=<from step 1>
OSM_CLIENT_SECRET=<from step 1>
OSM_REDIRECT_URI={{ status.redirect_uri }}
FLASK_SECRET_KEY=<any random string>
FERNET_KEY=<generate with the command below>

Generate a Fernet key:

python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"

Don't lose FERNET_KEY: changing it makes any previously stored OSM tokens unreadable, and you'll have to re-authorize.

3. Restart the server and authorize

The app reads .env at startup, so restart python run.py after editing it. Then click Authorize with OSM above — you'll be sent to OSM, approve the scopes, and bounced back to {{ status.redirect_uri }} with a success flash.

Switching between dev and prod

The DEV/PROD badge in the header reflects OSM_API_BASE. To switch, set the env var inline (it wins over .env because loading uses setdefault):

# PowerShell (prod)
$env:OSM_API_BASE="https://api.openstreetmap.org"
$env:OSM_CLIENT_ID="<prod-client-id>"
$env:OSM_CLIENT_SECRET="<prod-client-secret>"
python run.py
